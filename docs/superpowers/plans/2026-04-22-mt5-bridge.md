## Task 7: ClaudeBridge.mq5 EA

**Files:**
- Create: `.claude/profiles/ftmo/mt5_ea/ClaudeBridge.mq5`

Estructura (~250 líneas MQL5):
- Inputs: HeartbeatSec=5, Magic=77777, CommandsFile, StateFile, AllowExecution=true
- OnInit: `EventSetTimer(HeartbeatSec)`, Print versión + magic
- OnTimer: `ProcessCommands()` then `WriteState()`
- ProcessCommands: read JSON via FileOpen/FileReadString, parse manually (MQL5 no tiene JSON nativo pre-MT5-build-3630 — implementar parser simple o usar `<JAson.mqh>` include)
- ExecuteCommand: switch en type, construye MqlTradeRequest, llama OrderSend
- WriteState: BuildAccount + Positions + Pending + ClosedToday (HistorySelect con fechas de hoy)
- Idempotencia: skip si `processed==true`
- Error: si ExecuteCommand fails, `result.ok=false` con retcode

**JSON parser:** incluir `<Files/JAson.mqh>` si disponible, o parser básico custom (menos robusto pero sin dep externa).

Decisión: parser custom para no requerir librería externa. Las estructuras son planas, factible con parser tokenizer ~80 líneas.

- [ ] Commit: `feat: ClaudeBridge.mq5 EA (MT5 bridge)`
